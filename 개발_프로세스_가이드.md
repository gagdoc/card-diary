# Card Diary — 전체 개발 프로세스 가이드

> 마지막 업데이트: 2026-04-20

---

## 1. 프로젝트 개요

**Card Diary**는 월별 카드 형태로 일기를 작성하고 관리하는 웹 앱입니다.
Google 계정으로 로그인하면 일기 데이터와 이미지가 **Google Drive에 자동 동기화**되어, PC와 모바일(아이폰 등) 어디서든 동일한 데이터를 이용할 수 있습니다. PWA(Progressive Web App)로 제작되어 홈 화면에 추가하면 앱처럼 사용 가능합니다.

---

## 2. 기술 스택

| 분류 | 기술 |
|---|---|
| 프레임워크 | React 19 |
| 번들러 | Vite 7 |
| 스타일링 | Tailwind CSS 3 |
| 애니메이션 | Framer Motion |

| 코드 품질 | ESLint 9 |
| 배포 | Vercel |

## 4. 로컬 개발 환경 세팅

### 4-1. 사전 준비

- **Node.js** 설치 필요 (v18 이상 권장)
- **npm** 사용 (package-lock.json 기준)

### 4-2. 의존성 설치

```bash
npm install
```

### 4-3. 환경변수 설정 (`.env` 파일)

프로젝트 루트에 `.env` 파일을 만들고 아래 내용을 입력합니다.
이 파일은 **절대 GitHub에 올리면 안 됩니다** (`.gitignore`에 포함되어 있음).

```env
VITE_GOOGLE_API_KEY=your_google_api_key
VITE_GOOGLE_CLIENT_ID=your_google_client_id.apps.googleusercontent.com
```

> Google Cloud Console에서 발급받은 값을 넣어주세요. (아래 섹션 참고)

### 4-4. 개발 서버 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:5173` 으로 접속하면 앱이 실행됩니다.

Vite의 **HMR(Hot Module Replacement)** 덕분에 코드를 수정하면 브라우저가 자동으로 반영됩니다.

### 4-5. 개발 중 API 프록시

`vite.config.js`에 개발 서버 프록시가 설정되어 있습니다.

```
/api/google → https://photoslibrary.googleapis.com
```

로컬에서 Google Photos API를 호출할 때 CORS 문제를 피하기 위한 설정입니다.

## 8. Vercel 배포

### 8-1. 첫 배포 (초기 설정)

1. [vercel.com](https://vercel.com)에서 GitHub 계정으로 로그인
2. **Add New Project** → GitHub 저장소 선택(Import)
3. Framework Preset이 **Vite**로 자동 인식되는지 확인
4. **Environment Variables** 항목에 `.env` 파일의 내용을 직접 입력:
   - `VITE_GOOGLE_API_KEY` = `your_google_api_key`
   - `VITE_GOOGLE_CLIENT_ID` = `your_client_id`
5. **Deploy** 클릭 → 배포 완료 후 `https://your-project-name.vercel.app` 도메인 발급

### 8-2. 이후 배포 (자동)

GitHub에 코드를 `push`하면 Vercel이 자동으로 빌드·배포합니다. 별도의 작업이 필요 없습니다.

### 8-3. SPA 라우팅 설정 (vercel.json)

React는 클라이언트 사이드 라우팅을 사용하므로, URL을 새로고침할 때 404가 뜨지 않도록 `vercel.json`에 설정이 포함되어 있습니다.

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

## 9. Google Cloud Console 설정

Google 로그인과 Drive API를 사용하기 위해 한 번만 설정하면 됩니다.

### 9-1. 프로젝트 생성

1. [console.cloud.google.com](https://console.cloud.google.com) 접속
2. 새 프로젝트 생성

### 9-2. API 활성화

아래 API를 검색하여 **사용 설정(Enable)**:

- Google Drive API
- Google Docs API (선택)
- Google Calendar API (선택)
- Photos Library API (선택)

### 9-3. OAuth 2.0 클라이언트 ID 발급

1. **API 및 서비스 > 사용자 인증 정보 > OAuth 2.0 클라이언트 ID 만들기**
2. 유형: **웹 애플리케이션**
3. **승인된 자바스크립트 원본 (Authorized JavaScript origins)** 추가:
   - 개발용: `http://localhost:5173`
   - 배포용: `https://your-project-name.vercel.app`
4. **승인된 리디렉션 URI (Authorized redirect URIs)** 추가:
   - `https://your-project-name.vercel.app`
5. 저장 후 발급된 **클라이언트 ID**를 `.env`와 Vercel 환경변수에 등록

### 9-4. API 키 발급

1. **사용자 인증 정보 > API 키 만들기**
2. 발급된 **API 키**를 `.env`와 Vercel 환경변수에 등록

> **주의**: 배포 후 Google OAuth 설정에 Vercel 도메인 추가를 잊으면 로그인이 실패합니다. 도메인 변경 후 반영까지 최대 1시간이 걸릴 수 있습니다.

## 10. 데이터 저장 구조

### 인증 흐름

```
사용자 로그인 (Google OAuth)
  └─ access_token 발급
       ├─ localStorage에 저장 (세션 유지)
       └─ Google Drive API 호출에 사용
```

### 데이터 동기화 흐름

```
앱 실행 & 로그인
  └─ Google Drive에서 diary_entries.json 로드
       └─ 없으면 localStorage 데이터 사용

일기 작성/수정/삭제
  └─ 즉시 localStorage에 저장 (캐시)
  └─ 1초 디바운스 후 Google Drive에 자동 저장
       ├─ 이미지(base64): Drive의 CardDiaryImages 폴더에 업로드
       │    └─ 업로드 완료 후 { type: 'drive', id: fileId } 형태로 변환
       └─ JSON 데이터: Drive의 CardDiaryData/diary_entries.json 에 저장
```

### Drive 폴더 구조

```
Google Drive (사용자 계정)
├── CardDiaryData/
│   └── diary_entries.json    # 전체 일기 데이터 (JSON)
└── CardDiaryImages/
    ├── img_1234567890.png
    └── img_0987654321.png    # 업로드된 이미지 파일들
```

### 게스트 모드

Google 로그인 없이도 **Guest Access** 버튼으로 앱을 체험할 수 있습니다.
게스트 모드에서는 데이터가 브라우저의 **localStorage에만 저장**되고, Drive 동기화는 되지 않습니다.
