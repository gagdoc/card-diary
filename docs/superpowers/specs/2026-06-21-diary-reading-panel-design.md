# 다이어리 읽기 패널 설계

## 목표

월별 목록에서 일기를 클릭했을 때 편집 화면을 즉시 열지 않는다. 사용자가 작성한 날짜, 제목, 본문과 모든 사진을 먼저 읽기 전용 상세 패널에서 보여주고, 명시적으로 `수정`을 눌렀을 때만 기존 편집 화면으로 이동한다.

## 사용자 흐름

1. 사용자가 연도 화면에서 월 카드를 선택한다.
2. 오른쪽 패널에 해당 월의 일기 목록이 열린다.
3. 목록의 일기를 선택하면 같은 패널이 일기 상세 화면으로 전환된다.
4. 상세 화면에서 날짜, 시간, 위치, 제목, 본문 전체와 저장된 사진을 확인한다.
5. 상단의 `← M월 목록`을 누르면 기존 월 목록으로 돌아간다.
6. `수정`을 누르면 선택한 일기의 기존 편집 화면이 열린다.
7. 저장 또는 취소 후에는 선택했던 일기의 상세 읽기 화면으로 돌아온다.
8. 월 목록의 `새 일기 작성`은 지금처럼 새 일기 작성 화면을 연다.

## 화면 구성

### 월 목록

현재 `MonthDetail`의 월별 목록과 대표 이미지 지정 기능을 유지한다. 일기 행 클릭의 목적지만 상세 읽기 화면으로 변경한다.

### 일기 상세

- 같은 오른쪽 패널 안에서 월 목록을 대체한다.
- 상단에는 월 목록 복귀 버튼, 날짜와 `수정` 버튼을 배치한다.
- 사진이 있으면 선택된 사진 한 장을 크게 표시하고, 여러 장이면 하단 썸네일과 좌우 이동 조작을 제공한다.
- 제목과 본문은 편집 컨트롤 없이 읽기 전용으로 표시한다.
- 시간과 위치는 값이 있을 때만 보조 정보로 표시한다.
- 사진이 없으면 사진 영역을 숨긴다.
- 제목이 없으면 `제목 없는 일기`를 표시하고, 본문이 없으면 빈 본문을 위한 짧은 대체 문구를 표시한다.

### 편집 화면

기존 `DiaryEditor`의 작성·편집 기능을 재사용한다. 상세 화면에서 전달된 일기만 편집 상태로 열고, 저장 또는 취소 시 App의 이동 상태를 통해 해당 일기의 상세 화면으로 복귀한다.

## 컴포넌트와 상태

### `MonthDetail`

- 월 목록 내에서 현재 선택한 일기 ID를 관리한다.
- 선택된 일기가 없으면 목록을, 있으면 `DiaryDetail`을 렌더링한다.
- 상세 화면의 복귀, 수정 요청을 상위 컴포넌트에 전달한다.
- 항목 삭제나 데이터 동기화로 선택된 일기가 사라지면 안전하게 월 목록으로 복귀한다.

### `DiaryDetail`

- 새 읽기 전용 컴포넌트로 분리한다.
- 일기 객체와 복귀·수정 콜백을 입력으로 받는다.
- 선택된 사진 인덱스를 내부 상태로 관리한다.
- 일기가 바뀌면 사진 인덱스를 첫 번째 사진으로 초기화한다.
- Drive 이미지와 로컬 이미지는 기존 `DriveImage`를 사용해 동일하게 표시한다.

### `App`

- 새 일기 작성과 기존 일기 편집을 구분한다.
- 상세에서 수정 요청을 받으면 선택한 일기를 편집 화면에 전달한다.
- 저장 또는 취소 후 월 패널의 동일한 일기 상세로 돌아갈 수 있도록 이동 상태를 보존한다.

### `DiaryEditor`

- 진입 즉시 편집해야 하는 경우와 새 일기 작성의 기존 역할을 유지한다.
- 현재 미완성 상태인 “선택 일기로 스크롤·강조” 변경은 제거하고, 명시적 편집 진입 방식으로 정리한다.
- 상세 화면으로 돌아가기 위한 완료·취소 콜백을 호출한다.

## 오류 및 경계 조건

- 이미지 배열과 과거 `imageUrl` 데이터를 모두 지원한다.
- 손상되거나 불러오지 못한 이미지는 기존 `DriveImage`의 오류 처리에 맡기고 나머지 콘텐츠는 계속 표시한다.
- 사진이 0장 또는 1장이면 불필요한 갤러리 이동 조작을 숨긴다.
- 선택 중인 일기가 삭제되거나 사라지면 상세 화면을 닫고 월 목록으로 복귀한다.
- 날짜, 제목, 본문, 위치 등 선택 필드가 없어도 상세 화면은 렌더링된다.

## 검증

- 월 목록에서 일기 클릭 시 편집 폼 대신 읽기 상세가 열린다.
- 상세 화면에 선택한 일기의 제목, 본문과 전체 사진이 표시된다.
- 썸네일 및 좌우 조작으로 선택 사진이 바뀐다.
- 월 목록 복귀 버튼이 같은 월 목록을 복원한다.
- `수정`으로 편집 화면에 진입하고 저장·취소 후 같은 일기의 상세 화면으로 돌아온다.
- 사진이나 텍스트가 없는 일기도 오류 없이 표시된다.
- 기존 새 일기 작성과 월 대표 이미지 지정 동작이 유지된다.
- ESLint와 프로덕션 빌드가 성공한다.

## 범위 제외

- 전체 화면 사진 확대 뷰어
- 댓글, 공유, 좋아요 기능
- 사진 순서 편집
- URL 기반 라우팅 도입
