# 2026-06-21 개발 히스토리

이 문서는 2026년 6월 21일에 적용한 변경과 다음 개발자가 작업을 이어가기 위해 알아야 할 결정 사항을 기록한다.

## 최종 사용자 흐름

```text
연도 화면 → 월 카드 → 월 일기 목록 → 일기 읽기 상세
                                      ├─ M월 목록 → 월 목록 복귀
                                      └─ 수정 → 편집기 → 저장/취소 → 같은 상세 복귀
```

- 월 목록에서 일기를 클릭하면 곧바로 수정하지 않고 같은 오른쪽 패널에서 읽기 상세를 연다.
- 상세에는 날짜, 제목, 본문 전체, 작성 시간, 위치와 모든 사진을 표시한다.
- 여러 사진은 큰 이미지, 썸네일, 이전/다음 버튼, 좌우 스와이프로 탐색한다.
- 상세의 `수정`을 눌렀을 때만 기존 `DiaryEditor`를 연다.
- 저장 또는 취소 후 `MonthDetail`의 선택 ID를 유지해 같은 일기의 최신 상세로 돌아간다.

## 오늘 적용된 주요 업데이트

### 일기·사진 UX

- `DiaryEditor` 다중 이미지 및 Google Drive 이미지 선택 흐름 개선
- 월 대표 이미지 선택 기능과 이미지 상단 기준 표시
- 앱 시작 시 현재 월 자동 스크롤
- 읽기 전용 `DiaryDetail` 컴포넌트 추가
- 월 패널이 열렸을 때 전역 새 일기 버튼을 숨겨 조작부 겹침 방지
- 목록·수정·사진 탐색 버튼의 키보드 접근, ARIA 이름, 포커스 표시 보강

### Google 인증·동기화

- Google Drive 401을 토큰 만료로 분류하고 재인증 흐름으로 전달
- 인증 상태를 `authenticated`, `needs_reauth`, `unauthenticated`로 분리
- 사용자 정보는 유지하고 만료 토큰만 정리하는 로그인 지속 방식 적용
- Google Photos Picker 오류 안내 개선
- 로컬 개발 origin을 `http://localhost:5174`로 고정

### iOS 안정성

- 저장 전 사진을 최대 1920px, JPEG 75%로 압축
- LocalStorage 저장 시 base64를 제거해 iOS Safari 용량 초과 방지
- 전역 Error Boundary로 흰 화면 대신 복구 UI 제공

## OAuth 개발 환경 주의사항

- 개발 서버는 반드시 `npm run dev`로 실행한다.
- 주소는 `http://localhost:5174`여야 한다.
- `127.0.0.1:5174`는 Google OAuth에서 별개의 origin이며 `origin_mismatch`가 발생한다.
- `vite.config.js`의 `host`, `port`, `strictPort`를 임의로 변경하지 않는다.
- 포트를 바꾸려면 Google Cloud Console의 승인된 JavaScript 원본도 정확히 같은 origin으로 등록해야 한다.
- Client secret이나 access token을 코드, 로그, 문서에 남기지 않는다.

## 코드 구조와 상태 책임

| 파일 | 책임 |
|---|---|
| `src/App.jsx` | 월 패널과 편집기 오버레이 전환 |
| `src/components/diary/MonthDetail.jsx` | 월 목록과 선택 상세 전환, `selectedEntryId` 관리 |
| `src/components/diary/DiaryDetail.jsx` | 읽기 전용 본문·메타데이터·사진 갤러리 |
| `src/components/diary/DiaryEditor.jsx` | 신규 작성과 명시적 집중 편집 |
| `src/components/diary/diaryDetailState.js` | 이미지 정규화, 빈 텍스트 대체, 사진 인덱스 순환 |
| `src/context/AuthContext.jsx` | OAuth 토큰 수명과 3단계 인증 상태 |
| `src/context/DiaryContext.jsx` | 계정별 캐시, CRUD, Drive 동기화 |

`MonthDetail`은 선택한 일기 객체를 복사해 저장하지 않는다. `selectedEntryId`만 보관하고 Context의 최신 `monthEntries`에서 다시 찾는다. 이 원칙이 수정 저장 후 상세 내용이 즉시 갱신되는 핵심이다.

## 데이터 호환성

- 새 데이터는 `images[]`를 우선 사용한다.
- 과거 데이터에 `imageUrl`만 있으면 단일 사진으로 표시한다.
- 제목이 없으면 `제목 없는 일기`, 본문이 없으면 `작성된 내용이 없습니다.`를 화면에만 표시한다.
- 대체 문구를 원본 데이터에 저장하지 않는다.
- 한 일기당 사진은 최대 5장이다.
